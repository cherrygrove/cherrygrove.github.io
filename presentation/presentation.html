CherryGrove b&b


A bread and breakfast website


Created by Colin McIllece  and Nick Poelma

What It Is

  • A website for the CherryGrove bed and breakfast to find customers online
  • A registering platform for the CherryGrove B&B to allow guests to register online
  • An informational brochure for customers interested in the CherryGrove B&B

Colin'S WORK

Front-End Development

  • Jade - A Whitespace-Sensitive Dynamic Templating Language for HTML 
  • LESS - A CSS Preprocessor with variables, mixins, and nesting 
  • jQuery -  A fast, small, and feature-rich Javascript library.  

Infrastructure Management

  • Git Version Control and GitHub Integration - A way to ensure all development code is consistent.
 
  • Github Pages - A hosting solution catered specifically to the Github collaboration tool, which allows for easy deployment.
  • HarpJS - A way to ensure that all Jade and LESS would be preprocessed without hassle, and allows for dynamic content powered by JSON files.

Backend Engineering

  • Firebase BaaS - A butt-based document-oriented backend with a AngularJS API, used to keep track of book and ticket orders
  • JavaScript and AngularJS API ensured that data in my butt perfectly synced with AngularJS $scope(like variables) in realtime.

Graphic Design

  • The use of sophisticated graphic editing software such as Adobe Photoshop and Fireworks to improve and enhance every graphic on the website. Tools such as cropping, filters, and even fabricating images from scratch.

Content Management/IA

  • Content Management - Development of advanced grammar, deployed in such a way that the text will invite users to remain on a website longer and augment the buyer conversion rate(number of users who visit a site and buy a product or complete a task).


  • Information Architecture - Involves the categorization of information into a coherent structure, preferably one that the intended audience can understand quickly, if not inherently, and then easily retrieve the information for which they are searching. Aesthetics and usability are key to a successful website.

Quality Assurance And

Test-Driven-Development

  • Quality Assurance - The process of executing the functions on the website with the intent of finding software defects or bugs in order to improve the user experience and allow easier implementation of code.
  • Test-Driven-Development - A process involving the setup of code tests, execution of the tests, validation of the code, and cleanup of the code to ensure that only the bare minimum of code needed is included in the files.
